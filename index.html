Data Types in Python & MySQL

Exercise 1 – Identify Python Data Types

a = 42
b = "Hello"
c = 3.14
d = True

1. What is the type of a?

Correct! a is an integer because it's a whole number.

2. Is b a string or integer?

Correct! b is a string because it's enclosed in quotes.

3. Which variable holds a floating-point number?

Correct! c is a float because it has a decimal point.

4. What is the value of d?

Correct! d is explicitly set to True.

Exercise 2 – Identify MySQL Data Types

SET @age = 25;
SET @name = 'Alice';
SET @price = 9.99;
SET @is_active = TRUE;

1. What data type is @age?

Correct! @age is an integer because it's a whole number.

2. Is @name a VARCHAR or INT?

Correct! @name is a VARCHAR (string) because it's text in quotes.

3. Which variable is a FLOAT?

Correct! @price is a FLOAT because it has decimal places.

4. How is @is_active stored in MySQL?

Correct! MySQL stores BOOLEAN as TINYINT(1) where 1 is TRUE and 0 is FALSE.

Exercise 3 – Type Errors in Python

x = "5"
y = x + 2  # What happens here?

Why does this fail?

Correct! Python is strongly typed and won't implicitly convert between strings and numbers.

Fix:

y = int(x) + 2  # Explicit conversion.

Exercise 4 – Type Casting in MySQL

SELECT '10' + 5;  -- What is the result?

What will this return?

Correct! MySQL weakly types and implicitly converts '10' to INT before adding.

Data Type Comparison

Category Python MySQL

Feature Python (Dynamic) MySQL (Static)

Flexibility High (no type declarations) Low (fixed column types)
Example x = 10 (type inferred) INT age (explicit)

Review Questions

MySQL INT is statically typed (fixed at declaration). Python int is dynamically inferred (can reassign to other types).

MySQL weakly types → converts '5' to INT. Python strongly types → requires explicit conversion (int('5') + 2).

Use JSON or a separate table with foreign keys.

The JSON data type (stores key-value pairs).

Case Studies

Python Dictionary

product = {
    "name": "Laptop",
    "price": 999.99,
    "in_stock": True
}

1. What is the type of product?

Correct! product is a dictionary (dict) because it uses key-value pairs.

2. How would you access the price?

Correct! Use product["price"] to access the value.

MySQL Table

CREATE TABLE users (
    id INT PRIMARY KEY,
    username VARCHAR(50),
    is_premium BOOLEAN
);

1. What type is is_premium?

Correct! is_premium is a BOOLEAN (stored as TINYINT(1)).

2. How would you insert FALSE?

INSERT INTO users (id, username, is_premium) VALUES (1, 'Alice', );

Correct! Use FALSE or 0 to insert a false value.
